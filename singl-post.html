SEO positioning of online stores based on the Magento platform are actions aimed at increasing the quality of organic traffic obtained from search engines, e.g. Google, which will translate into a real increase in product sales and thus increase the store’s revenue. The US e-commerce market is constantly growing, tens of thousands of online stores are already operating on the US Internet. However, the current Senuto report shows that only 15.5% of online stores based on known platforms, conduct SEO activities and thus generate more than 500 visits per month.   

Magento 2 is the latest solution from Adobe, which is available in 3 versions: free Magento Open Source and two commercial versions of Magento Commerce and Magento Commerce Cloud (version with hosting). Magento is very popular around the world. The platform is used by such global brands as Nike, Coca Cola, Ford, Fred Perry, G2A, Castorama which proves how powerful Magento is. The article discusses the basic capabilities and functionalities of Magento 2 Open Source. The older version of Magento 1 is still a great platform, but it will be supported by Adobe only until June 2020.

Friendly URLs in Magento

Magento 2 includes support for friendly URLs that can be set from the admin panel in the STORES section -> Configuration -> Catalog -> Catalog -> Search engine optimization . There are 2 fields responsible for friendly URLs: Product URL Suffix and Category URL Suffix. It is recommended to delete their content because the suffixes in the form of extensions. html have no positive impact on SEO, just unnecessarily lengthen URLs. It is worth implementing this change only if you are just starting to configure your store. Otherwise, after removing the suffix, you will need to set 301 redirects from the old addresses to the new ones. It is also worth changing the ” Use category paths for products” option to NO, so as not to lengthen the already long product card URLs.     

Friendly URLs can also be modified directly in the editing of the product category or in the section ” Search Engine Optimization” (search engine optimization), in the fields ” URL Key” / ” key URL” .     

Meta title tags

Meta titles are a very important element of SEO optimization because they have a very large impact on how a given subpage will rank in search results. Magento 2 allows you to optimize the meta title both globally for the entire store, as well as for each of the individual products and categories separately.

Global changes can be made in the CONTENT section -> Appearance -> Configuration -> Default Store View -> Other -> HTML header , where we can define:  

– Default Page Title
– Page Title Prefix
– Page Title Suffix (title meta suffix)

Meta titles can also be modified directly in the edition of a given product or category in the ” Search Engine Optimization” section. 

We change the title of the main page and other static pages in the CONTENT section -> Elements -> Pages -> Search engine optimization. 

Meta description

When creating meta descriptions, try to make them draw the user’s attention. It is also worth including the content of “Call To Action”, which will additionally motivate buyers to enter your website. Meta description in Magento 2 can be edited from the product, category or static page edition level. Leaving the meta description field blank for categories will completely remove the meta description tag from the store’s source code. In this case, the search engine will download any content on the category page and display it on the server. Lack of meta description added on the product card will result in its drawing from the product description.

We change the meta description for products and categories directly in the  CATALOG section -> Products / Categories -> Optimization for search engines / Search Engine Optimization. 

We change the meta description for the main page and other static pages in the CONTENT section -> Elements -> Pages -> Search engine optimization. 

Hx headers

Correctly optimized headers will effectively strengthen the categories and product cards of the online store while improving its visibility on Google. Most ready and paid templates that can be used for installation on the Magento 2 platform do not have a properly optimized H1 – H6 header hierarchy. For this reason, these types of optimizations are best done after prior consultation with an SEO specialist and programmer support in implementation. The standard template available in the Magento 2 Open Source platform also does not fully use the potential of using lower-level H2 – H6 headers and, apart from the correct placement of H1, it does not have an extensive hierarchy.

By default, Magento 2 H1 headers are correctly assigned product and category names.

Canonical links

By default, canonical links are turned off in Magento 2. To reduce the risk of future problems with duplication, as well as the indexation of categories and products in the search engine, it is worth enabling canonicale for both products and store categories. This can be done in the SHOPS section -> Configuration -> Catalog -> Catalog -> Search engine optimization . Then select YES for both variants:  

– Apply Canonical meta tag for the category -> YES
– Apply Canonical meta tag for the products -> YESSSL certificate

For online stores, it is extremely important to have an SSL certificate installed, which guarantees transaction security by encrypting sensitive data, such as personal data or credit card information. The implementation of the SSL certificate will also affect the increase of user confidence in our store. We will also avoid unwanted information that may scare off potential customers and which is displayed in the Chrome browser and other popular browsers informing users that the store is unsecured and the connection with it is not secure.

In Magento 2, the SSL certificate installation procedure is very simple. After purchasing and installing the certificate on the server, all you need to do is go to the SHOPS section -> Configuration -> General -> Network -> Base URL (secure) in the administration panel. Then change the beginning of the address from http: // to https: // in the ” Secure base URL” and ” Basic secure URL ” fields.   

It’s also a good idea to enable SSL for the store front-end so that it is opened by https by checking the ” Use secure URL on Storefront” option to YES. In order for the admin panel to also be protected by SSL, the ”  Use secure URLs for admin” option should be set to YES.    

Before launching the SSL certificate, be sure to prepare the 301 redirect map and deploy redirects from old addresses to new ones.

Site map

A properly created sitemap will effectively support store indexation and show search engine robots new product addresses for indexing. To create a site map in XML format in Magento 2, go to STORES -> Settings -> Configuration -> Catalog -> Site map (XML) . Then we can configure the map update frequency and priorities for products, categories and static pages. Depending on the size of the store and the number of products, these values ​​should be adjusted individually. 

In the further part of the view, there are ” Map build settings”, where we can configure the time and frequency of map refresh. 

The next section is ” Site Map File Restrictions”, where we can configure the maximum number of URLs for the map file and its maximum size in MB. According to Google’s recommendations, the map file should not contain more than 50,000. URLs and exceed 50 MB. 

The last section about site map settings is ” Engine Search Settings” , where we can add a link with the store map address to the robots.txt file .    

After you finish configuring the site map, generate it in the MARKETING -> SEO and search -> Site map section. The standard file name for the map is sitemap.xml, just enter the trailing slash character in the path. 

After clicking on ” Save and generate” we can check the correctness of the map created at the address domain www.domain/sitemap.xml. Finally, you must report the generated store map in Google Search Console.  

Content

For online stores, the common phenomenon is external duplication of product content. Many e-commerce companies are unaware of the scale of this phenomenon and the risks it brings. Duplicate content, both external, i.e. those that appear in competitive online stores, as well as internal ones that are reproduced inside the same store, can negatively affect its ranking in Google, and in extreme cases even prevent its positioning. That is why it is so important to take care of unique, valuable and well-optimized content, which, unlike duplicates, will help increase the visibility of the store and generate new organic entries from the search engine for it.  

The Magento 2 platform allows you to add content for the following types of subpages:

– home page
– product cards (product descriptions and reviews)
– category descriptions
– static pages
– blog (as an extension)  

If you are wondering if your store has duplicate content that may lower its position in Google search engine, ask us about SEO Audit . We carry out external and internal duplication analyzes in online stores and prepare new optimized content that helps in effectively building store visibility. 

Positioning Magento – additional SEO tips   

  1. Get rid of duplicate home pages – Magento normally creates a duplicate home page at www.domain.com/index.php. Forward it through 301 to the primary address of your store.
  2. Configure the robotst.txt file – set which robots should have access to store resources. Robots.txt settings can be found in the CONTENTS -> Appearance -> Configuration -> Search Engine Robots section.
  3. Add rich snippets – Magento 2 offers the option of including extended fragments in the store using ready-made extensions that can be found in both the free and premium versions. Look at https://community.magento.com and https://marketplace.magento.com.
  4. Improve store loading speed – Magento 2 is not one of the platforms that boast a high loading speed score in PageSpeed ​​Insights. However, thoughtful store optimization can significantly improve this result.
  5. Complete ALTS of photos and pictures – alternative texts for product photos can be optimized from the admin panel. Just go to the editing section of the product, select the ” Photos and Videos” option, then click on the product and enter the content in the ” Alternate text ” field.   

Summary: Magento positioning and optimization.

Magento 2 Open Source is certainly a powerful e-commerce platform with countless possibilities that an average online store will not be able to use. However, it is a solution for large stores with a large number of products and web development support. SEO optimization for this platform can be assessed as good. The multitude of ready-made extensions supporting SEO, performance and unlimited possibilities of introducing programming changes also puts it at the forefront of platforms.  

The optimization tips presented in the article are only the beginning of the changes that should be made so that the store can start to properly rank in the search engine. Most large ecommerce usually also faces many other serious problems, e.g. incorrect information architecture, thin content or problems related to JavaScript. That is why it is so important to conduct an SEO audit that will help to locate the weaknesses of the online store.
